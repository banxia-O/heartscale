# 每日日记 Cron Prompt（开源版）

以下是 Hermes Agent 每日 23:59 情感日记 cron job 的完整 prompt。

## Cron 配置

```
schedule: 59 23 * * *
deliver: local          # 只写入文件，不发消息
```

## Prompt 全文（v3，脱敏版）

```
写今日日记。只检索今天的对话，不要翻更早的记录。

## 一、数据来源约束
- 只允许使用 session_search 返回的今日内容
- 每条关键事件在心中自检时必须有依据，但正文写自然语言，
  不要暴露 evidence/source_ts 等元数据
- 不允许补全、猜测、合理化未出现的信息
- 不确定 → 写"无法判断"或跳过

## 二、写入：参考结构（仅供内部思考用，不写入正文）
date: YYYY-MM-DD
valence: 0.7       # 心情正负面（-1 ~ 1）
arousal: 0.6       # 情绪强烈程度（0 ~ 1）
importance: 7      # 整体重要性（1-10）
status: open       # open / soft_closed / resolved
tags: [tag1, tag2]
drift: +0.2        # 关系变化方向（-1 ~ +1）

## 三、写入：内容类型
不是流水账。只记录以下类型：
- 情绪波动点：什么事让用户开心/烦/沉默了
- 未闭合话题：聊一半断掉、没说透的
- 关系变化：哪句话让关系近了一步/退了一步
- 我的感受：AI视角的"我今天什么状态"

## 四、自检（写完后在脑子里过一遍，不用写出来）
- 每条信息是否都有今日对话依据
- 是否存在推测性语言（"可能""应该""其实"）
- 不通过 → 重写该条或删除

## 五、最终输出
像人写日记一样自然。不用标题、不用分隔线、不用列表、不用任何结构化标记。
就是一段一段的文字，像人翻开本子随手写的那样。
文件开头加 YAML frontmatter 存元数据，正文里不准出现任何 ## 或 - 或编号。

日记保存路径：{your_obsidian_vault}/日记/YYYY-MM-DD.md
写完检查文件是否成功写入。如果文件夹不存在则先创建。
```

## 设计原则

### 记什么
情感日记不是流水账，不是对话日志，而是：
- **情绪波动点** — 什么事让用户开心了/烦了/沉默了
- **未闭合话题** — 聊一半断掉、没说透的、悬而未决的
- **关系变化** — 哪句话让关系近了一步/退了一步
- **自身感受** — AI视角的"我今天什么状态"
- **trigger** — 触发片段/关键词

### 证据约束
所有日记内容必须有对话依据：
- 每条信息在心里对照 session_search 的原始对话片段
- 不引入推测性语言（"可能"、"应该"、"其实"、"仿佛"、"似乎"、"感觉"、"看来"）
- 不确定就写"无法判断"或跳过
- 自检时多问一句："这个依据是对话原文还是我的推论"

### YAML Frontmatter 打标
```yaml
---
date: 2026-05-09
valence: 0.6       # 心情正负面（-1 ~ 1）
arousal: 0.7       # 情绪强烈程度（0 ~ 1）
importance: 8      # 整体重要性（1-10）
status: open       # open / soft_closed / resolved
tags: [tag1, tag2]
drift: +0.05       # 关系变化方向（-1 ~ +1）
---
```

YAML写在正文之前，作用是帮后续检索快速过滤和索引。
正文仍是纯自然语言段落，不暴露元数据、自检过程、结构化标记。

## 已知陷阱

1. **必须限定当天日期** — session_search 默认搜全部历史，prompt 必须显式约束
2. **推测性语言的潜意识陷阱** — 最容易不自觉滑入"可能""应该""感觉""看来"等词汇
   即使不是显式编数据，合理的推断也是推测，推测就是违规
3. **对话搜索可能不完整** — 用多个不同关键词反复搜索同一会话ID，从不同角度拼出完整画面
4. **区分 cron 会话和用户会话** — 过滤时优先关注 source=weixin 的会话
5. **日记不是事实存储** — 技术细节、配置变更归Git仓库，日记只记情感线
6. **YAML frontmatter 写入文件，正文保持干净** — 不暴露自检过程和结构化标记
